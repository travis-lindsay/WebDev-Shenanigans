Game Object:

The values of the Game Object properties are determined by the user-selected difficulty.

The Game Object contains the following properties:

  • difficulty: String, The difficulty of the puzzle, (easy, medium, hard)
  • numToWin: Integer, the number of blank spaces of the puzzle. This is decremented each time a user places a correct number
  • puzzleId: Two dimensional array of integers. Contains the Answer-Key of how the completed puzzle should look
  • puzzleToLoad: Two dimensional array, containing the initial puzzle appearance, with 0's for the blank spaces that are loaded

Game Object Examples:

Sudoku Game Variables:

  • selectedNumber: Integer, represents the selected number of a row of buttons 1-9. While a certain number is selected, that is the number that will be placed on the grid when the individual cells are selected.
  • numHints: Integer, the number of currently requested hints by the user
  • numErrors: Integer, the number of accumulated errors the user has commited

Functions/Methods of the Model:

The following functions belong to the Model of the Sudoku Game

  • pickGame():
    This function will take the game difficulty value picked by the user, and create a game object with the associated values for the difficulty (easy, medium, or hard).
  • giveHint():
    This function will find a randomly generated blank spot in the sudoku grid, place a value in it, and increment the numHints variable.
  • checkValidNumber():
    This function will take the number just inserted into the cell of the table, and check it against the two dimensional array of the correct numbers for that particular puzzle.
  • correctNumber():
    This function is called if the inserted number is in the correct place. It decrements the number of cells left over until the game has been won.
  • wrongNumber():
    This function is called if the inserted number is in the incorrect place. It will discard the number, and increase the number of errors in the numCurErrors variable.
  • checkGameWin():
    This function is called from the correctNumber() function, to update and check the numToWin variable to see if the game is now complete.
  • checkGameLose():
    This function is called from the wrongNumber() function, to check the numErrors variable against the max NUMERRORS constant to see if the game is over.
  • gameWin():
    This function is called from the checkGameWin function if the game has been won. It takes the game statistics and resets the game variables, then calls a render function to render the game win overlay.
